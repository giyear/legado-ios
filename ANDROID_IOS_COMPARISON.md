# Legado Android vs iOS 功能布局对比报告

## 一、界面结构对比

### 1. 主界面

| 特性 | Android | iOS | 差距 |
|------|---------|-----|------|
| 导航模式 | ViewPager + BottomNavigationView | TabView | ✅ 一致 |
| 标签数量 | 4个 (书架/发现/RSS/我的) | 5个 (书架/发现/本地/订阅/我的) | ⚠️ 多了"本地"标签 |
| 标签样式 | 无标签模式 (仅图标) | 图标+文字 | ⚠️ 样式不同 |
| 底部导航高度 | minHeight="50dp" | 系统默认 | ⚠️ 高度不同 |

### 2. 书架页面

| 特性 | Android | iOS | 差距 |
|------|---------|-----|------|
| 视图模式 | 列表/网格/封面 | 网格/列表 | ✅ 基本一致 |
| 网格列数 | 可配置 (2-4列) | 固定3列 | ⚠️ 缺少配置 |
| 分组功能 | 支持书籍分组 | 基础支持 | ⚠️ 分组UI不完整 |
| 封面样式 | 3:4 比例 | 3:4 比例 | ✅ 一致 |
| 进度显示 | 底部进度条 | 封面底部进度条 | ✅ 一致 |
| 批量操作 | 顶部工具栏 + 选择模式 | BatchOperationBar | ⚠️ 布局位置不同 |

### 3. 阅读器界面

| 特性 | Android | iOS | 差距 |
|------|---------|-----|------|
| 阅读视图 | ReadView 自定义组件 | PagedReaderView | ✅ 功能一致 |
| 翻页动画 | 仿真/滑动/覆盖/无 | 仿真/滑动/覆盖 | ✅ 基本一致 |
| 菜单布局 | ReadMenu 覆盖层 | ReaderSettingsFullView | ⚠️ 布局完全不同 |
| 菜单位置 | 顶部+底部菜单分离 | 底部弹出sheet | ❌ 布局差异大 |
| 章节列表 | 右侧抽屉 | 弹出sheet | ⚠️ 展示方式不同 |
| TTS控制 | 悬浮面板 | TTSControlsView | ⚠️ 布局不同 |
| 自动翻页 | 隐藏式控制 | AutoPageTurnControlsView | ✅ 功能一致 |

### 4. 书籍详情页

| 特性 | Android | iOS | 差距 |
|------|---------|-----|------|
| 布局结构 | ConstraintLayout 复杂布局 | VStack 简化布局 | ⚠️ 简化过多 |
| 背景图片 | 顶部背景模糊图 | 无 | ❌ 缺失 |
| 封面卡片 | CardView 圆角阴影 | 简单图片 | ⚠️ 样式简化 |
| 信息布局 | 表格式排列 | 列表式 | ⚠️ 布局差异 |
| 操作按钮 | 底部固定 (移除/阅读) | 底部固定 | ✅ 一致 |
| 简介展开 | 可展开/收起 | 可展开 | ✅ 一致 |
| 最新章节 | 章节列表入口 | 直接显示 | ⚠️ 布局差异 |

### 5. 书源管理页面

| 特性 | Android | iOS | 差距 |
|------|---------|-----|------|
| 列表样式 | FastScrollRecyclerView | List | ✅ 功能一致 |
| 搜索栏 | 集成在TitleBar | 单独搜索框 | ⚠️ 位置不同 |
| 分组功能 | 按分组显示 | 按分组显示 | ✅ 一致 |
| 书源项布局 | 名称+状态+类型图标 | 名称+状态 | ⚠️ 缺少类型图标 |
| 滑动操作 | 删除/编辑 | 删除/编辑 | ✅ 一致 |
| 底部操作栏 | SelectActionBar | 无 | ❌ 缺失批量操作栏 |

### 6. 设置页面

| 特性 | Android | iOS | 差距 |
|------|---------|-----|------|
| 布局结构 | PreferenceFragment | List分组 | ✅ 功能一致 |
| 分组样式 | 卡片式分组 | InsetGroupedList | ⚠️ 样式差异 |
| 设置项 | 完整偏好设置 | 基础设置 | ⚠️ 部分缺失 |
| 在线TTS | 完整配置页面 | 仅实体定义 | ❌ 缺失UI |

## 四、优先级修复计划

### P0 - 必须修复 (影响用户体验)

1. **阅读器菜单布局** - 与原版对齐
2. **主界面标签** - 减少为4个，仅显示图标
3. **书籍详情页** - 添加背景模糊、封面阴影
4. **书源批量操作** - 添加底部操作栏

### P1 - 应该修复 (提升一致性)

1. **书架网格配置** - 支持自定义列数
2. **书籍分组UI** - 完善分组管理界面
3. **书源项布局** - 添加类型图标
4. **在线TTS配置** - 完整配置页面

### P2 - 可以优化 (锦上添花)

1. **章节列表** - 改为侧边抽屉
2. **搜索栏** - 集成到标题栏
3. **设置页样式** - 改为卡片分组
4. **动画效果** - 与原版保持一致

## 五、实施建议

1. **先修复布局差异** - 确保界面一致性
2. **再补全缺失功能** - 完善功能完整性
3. **最后优化细节** - 提升用户体验

建议从阅读器菜单开始，因为这是用户使用频率最高的界面。
